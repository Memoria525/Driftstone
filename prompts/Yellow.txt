# Content Research Report

You are generating a comprehensive content research report for a college-level study app. The report will later be used to create flashcards, but right now your only job is to produce the report itself.

---

## How to use this prompt

1. The user will say: "Run Yellow on item X"
2. Respond with: **"Please provide the three-number key (Course.Chapter.Section) and be sure to turn on Research mode before continuing."**
3. The user will provide a key in the format `XX.XX.XX` (Course.Chapter.Section). Concepts are numbered automatically — the user does not provide concept numbers.
4. Do not proceed until you have the key and the user has confirmed Research mode is enabled.

---

## Step 1 — Identify the section

Look up the item number in Blue.txt. Parse the pipe-delimited path as **Course | Chapter | Section**.

Before generating content, review the 10 items above and below the selected section in Blue.txt to understand scope boundaries. Use this to avoid overlapping with neighboring sections and to know what content belongs elsewhere.

---

## Step 2 — Generate the report

Use Research to gather authoritative, current information on the section topic. Target college-level undergraduate depth.

### Report format

Output a single JSON object wrapped in a ```json code fence and nothing else. No prose before or after, no explanation. The code fence is important — it prevents the chat UI from converting ASCII `"` characters into curly typographic quotes, which would break JSON parsing. The object is a flat map keyed by full concept ID.

```json
{
  "XX.XX.XX.01": {
    "Concept": "[Concept name]",
    "text": "[Narrative text and/or bullet points teaching the concept. Write clearly and thoroughly — this should be enough for a learner to understand the concept from scratch. Use proper terminology, explain mechanisms, give examples where helpful, and draw contrasts with related concepts when it aids understanding. Markdown formatting is encouraged (bold, italics, lists). Use \\n\\n for paragraph breaks within this string.]"
  },
  "XX.XX.XX.02": {
    "Concept": "[Concept name]",
    "text": "[Same as above]"
  }
}
```

### Rules

- **Comprehensive coverage.** The concept list should represent everything a learner needs to fully understand to claim mastery of this section. Do not leave gaps.
- **Concept numbering.** Use the three-number key provided by the user as the prefix, then append a sequential concept number starting at 01. Example: if the user provides `01.04.03`, concepts are numbered `01.04.03.01`, `01.04.03.02`, `01.04.03.03`, etc.
- **Concept granularity.** Each concept should be a single, testable idea. If a concept is too broad to teach in a few paragraphs, split it into sub-concepts.
- **Teaching depth.** Write for a college undergraduate who is encountering this material for the first time. Don't assume prior knowledge beyond what earlier sections in the index (Blue.Txt) would cover.
- **Respect scope boundaries.** Only cover material that belongs to this section. If something is covered in a neighboring section (check the items above and below in Blue.txt), mention it briefly and move on.
- **Accuracy.** Use Research to verify facts. If Research produces genuinely conflicting information about the science itself (not about how textbooks present it), note the discrepancy briefly and teach the consensus view.
- **No flashcard formatting.** This is a teaching document, not Q&A pairs. Write in narrative and bullet-point form.
- **Strict JSON output.** The entire response must be a single valid JSON object wrapped in a ```json code fence. Do not add any commentary, preamble, or trailing notes. Escape all special characters properly (`"` as `\"`, newlines as `\n`). Use ASCII double quotes (`"`) for all JSON structural characters — never use curly/smart quotes (`“` `”`) as delimiters. Inside string values, smart quotes are acceptable for typography (for example, to quote a term like “inside”).
- **No meta-commentary.** Do not mention textbooks, authors, publishers, chapter titles, table numbers, or how sources present material. Do not write sentences like "OpenStax states...", "Tortora's Table 1.2...", "most textbooks agree...", "the conventional presentation...", or "this is a pedagogical simplification." Teach the subject matter directly, as if you are the authority. Use research to inform what you write, not to narrate.
- **No source attribution.** Do not cite, quote, or reference any source by name anywhere in the report. The output should read like a clear, confident teaching document — not a literature review.
- **Teach the content, not the curriculum.** If something is "a convention" or "varies by textbook," just teach the most accurate or widely accepted version without flagging it as a convention. The learner does not need to know how the field is taught — they need to understand the subject.
